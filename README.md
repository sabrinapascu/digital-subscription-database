# Digital Subscription Management Database

## Overview
Relational database system designed for managing digital subscriptions, including clients, subscription types, contracts, and payment history.

## Database Design
The schema is structured to ensure data integrity and avoid redundancy through normalization principles.

Main entities:
- CLIENTI
- TIPURI_ABONAMENTE
- ABONAMENTE
- PLATI

## Key Features
- Primary and foreign key relationships
- Data validation constraints
- Status management (ACTIV, EXPIRAT, ANULAT, PENDING)
- Financial tracking through payments

## Advanced SQL Operations
- JOIN, GROUP BY, HAVING
- EXISTS, UNION, MINUS
- MERGE statement
- Views and sequences
- Index creation for optimization

## Technologies
- Oracle SQL
- Relational database design
- Normalization
